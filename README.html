Відділ кадрів

База даних – SQLite 3

Містить 3 таблиці:

  1. teachers (викладачі)

    • id
    • name (ПІП)
    • gender (стать)
    • post (посада)
    • department (відділ)
    • bet (ставка)

  2. departments (відділи)

    • id
    • name (назва відділу)
    • is_bet_in_table (флаг, що вказує на те, що комірки таблиці будуть заповнятися часами - true, false - буквою «р»)
    • deputy_director (заступник директора відділа – ПІБ, вказується для Excel файлу)
    • deputy_director_with (посада заступника директора, вказується для Excel файлу)

  3. posts (посади)

    • id
    • name (назва посади)

Меню програми

  1. Файл

    • Новий місяць – відкриває меню вибора місяца для генерації місячних таблиць відділів, при цьому активні таблиці місяця будуть закриті.
    • Зберегти таблиці місяців .xslx – зберігає таблиці місяців по відділам в Excel файл. За замовчуванням файл називається місяцем, ім'я файлу можна змінити в меню вибору шляху збереження файлу. Цей елемент меню не активний, якщо таблиці місяців не згенеровано.
    • Вихід – закриття програми.

  2. Таблиці

    • Таблиці відділів – місячні таблиці відділів. Цей елемент меню не активний, якщо таблиці місяців не згенеровано.
    • Посади – таблиця БД для редагування/видалення та перегляду даних.
    • Відділи – таблиця БД для редагування/видалення та перегляду даних.
    • Співробітники – таблиця БД для редагування/видалення та перегляду даних.

  3. Допомога

    • README – відкриває .html інструкцію для користувача в браузері.

Опис роботи програми

  1. Програма може бути запущена тільки в одному екземплярі.
  2. Для корректної роботи програми з базою даних, потрібен дозвіл на запуск з правами адміністратора.
  3. При першому відкритті програми, користувачу показується меню зі списком вибору місяця для генерації таблиць по відділам.
  4. Таблиці «Посади», «Відділи», «Викладачі» можуть редагувати/видаляти об'єкти, натискаючи ПКМ на рядок таблиці і вибрав відповідну операцію «Змінити дані» або «Видалити». При натисненні на «Змінити дані» відкривається вікно, в якому можна редагувати поля об'єкта. Кнопка «Додати запис в Базу Даних» відкриває таке ж вікно, як і при зміні даних.

Таблиці відділів (місяців)

  1. «Таблиці відділів» мають n таблиць (залежить від кількості відділв), в яких по 31 день в місяці.
  2. При створенні ці таблиці автоматично заповнюють 15 перших днів місяця даними, в залежності від ставки співробітника. Також вказуються віхідні дні в залежності від кількості днів в місяці, наприклад, якщо в місяці 29 днів, то останні 2 дні (30 і 31) вказуються як вихідні.
  3. При першій ініціалізіції таблиці, якщо у викладача вказано ставку = 1, то в комірці дня виводиться буква р.
  4. Навігація по таблицям відбувається по виборі потрібної таблиці на панелі вкладок, де заголовок вкладки – назва відділу.
  5. Тип дня змінюється при натисканні ПКМ на заголовок колонки дня: робочий, вихідний, святковий, передсвятковий. Вихідний тип змінює фон комірок дня цієї колонки.
  6. Контекстне меню відкриваються ПКМ на рядок таблиці. Якщо клік відбувся на комірці дня, то меню має 3 елементи: «Редагувати ячейку», «Змінити тип днів в діапазоні» і «Видалити рядок», якщо на любій другій комірці – «Змінити тип днів в діапазоні» і «Видалити рядок».
  7. Пункт «Редагувати ячейку» в контекстному меню відкриває форму редагування поля, в якому можна вказати або ставку викладача, або зробити рядок пустим, або обрати тип дня, які потім в таблиці позначаються однією буквою: Робочий (р), Відпустка (в), Лікарняний (л), Відрядження (В).
  8. Пункт «мінити тип днів в діапазоні» в контекстному меню відкриває форму вводу діапазону для зміни типу днів. Вводити в поля діапазону дозволено числа [1, 31].
  10. Колонки «Відпрацьовано» днів/годин рахуються при зміні даних таблиці.
  11. Кількість днів рахуються по робочим дням викладача, якщо він не на лікірняному або у відпустці, та вказаним типам дня – робочий та передсвятковий.
  12. Кнопка «Додати співробітника» відкриває форму, в якій є список з новими співробітниками відділу, яких можна додати в таблицю. Співробітники, які вже є в таблиці в цей список не попадають. Якщо всі можливі співробітники вже в таблиці, виводиться відповідне повідомлення.
  13. Таблиці місяців зберігаються в файл при закритті програми та кожні 3 хвилини роботи програми. При повторному відкритті програми вони ініціалізуються.
  14. При створенні нового місяця, дані старого місяця зберігається в теці history, в місці, де встановлена програма, для того, щоб була можливість відновити дані старого місяця в програмі при необхідності. Щоб відновити дані, потрібно обрати потрібний файл в цій теці і замінити цим файлом файл months_data в корні програми.
